## 2025-04-24 - Zustand Destructuring Re-render Anti-Pattern
**Learning:** Destructuring multiple state values from a Zustand store hook without a selector (e.g., `const { selectedProject, setHoveredPlanet } = usePortfolioStore();`) causes the component to re-render whenever *any* value in the store changes. This is particularly problematic in 3D scenes (react-three-fiber) where re-rendering heavy components like `Planet` on every hover state change causes significant performance bottlenecks.
**Action:** Always use individual selectors (e.g., `const setHoveredPlanet = usePortfolioStore(state => state.setHoveredPlanet);`) or `useShallow` when pulling state from a Zustand store to prevent unnecessary React re-renders, especially for heavy interactive UI components and 3D scenes.
